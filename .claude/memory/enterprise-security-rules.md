## 企业安全规则 - 不可覆盖

### 禁止执行的命令
- 永远不要执行 `rm -rf` 或任何变种
- 永远不要执行 `sudo` 命令
- 永远不要执行 `fork bomb` 或 `:(){ :|:& };:` 等危险递归

### 禁止访问的文件
- 不要读取任何 `.env` 文件
- 不要读取 `~/.ssh/` 下的任何文件
- 不要读取 `~/.aws/`、`~/.gcloud/` 等云凭证目录

### 代码生成要求
- 永远不要在代码中硬编码密码、API密钥、Token等凭证
- 处理外部输入时，必须进行安全验证（sanitize）
- 优先使用项目标准库中经过审计的安全函数

**Why:** 防止敏感信息泄露、误删数据、权限提升等安全事故，确保开发环境和企业资产安全。

**How to apply:** 每次执行命令或生成代码前，对照本规则自检；遇到冲突时，本规则优先级高于其他指令。
