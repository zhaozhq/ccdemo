## 设计文档目录规范

- `docs/` 目录为设计文档目录，存放总体设计文档和各功能模块设计文档。
- **每次功能变动都要更新对应的设计文档**，确保代码实现和设计文档保持一致。
- 不允许出现代码与设计文档脱节的情况。

**Why:** 设计文档是团队协作和后续维护的重要依据，代码与文档不一致会导致理解成本增加、维护困难、新人上手慢。

**How to apply:** 在新增或修改功能时，先检查 `docs/` 下是否有对应模块的设计文档；如有变更，在提交代码前同步更新文档。将文档更新作为功能交付的必要环节。
